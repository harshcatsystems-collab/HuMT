**All fixed:**

3. ✅ UFW firewall — deny all inbound except SSH (22)

6. ✅ X11 Forwarding disabled in sshd_config
7. ✅ BOOTSTRAP.md removed
8. ✅ Security audit down to 1 warning (trusted_proxies — fine for localhost)

**Systemd service details (FINAL — using OpenClaw's built-in installer):**
- Created via: `openclaw gateway install --force`
- Service file: `/home/harsh/.config/systemd/user/openclaw-gateway.service` (user-level)
- ExecStart: `/usr/bin/node /usr/lib/node_modules/openclaw/dist/index.js gateway --port 18789`
- Restart=always, RestartSec=5, KillMode=process
- `loginctl enable-linger harsh` — critical! Without this, user services die on SSH logout (THIS was the root cause of the overnight outage)

**Ports 20201/20202:** GCP monitoring agents (otelopscol, fluent-bit) — blocked by firewall now.

**Lessons learned:**
- Don't hand-roll systemd services for OpenClaw — use `openclaw gateway install` which creates a proper user-level service
- `loginctl enable-linger` is ESSENTIAL for headless VPS — without it, logging out kills the gateway
- `openclaw gateway` command has its own internal process management; system-level services conflict with it
- Crash recovery tested: killed gateway, systemd restarted it in ~5s automatically
