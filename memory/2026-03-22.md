## 07:25 UTC — Additional Autoresearch Content

### PRD Evolution Post (Aakash Gupta)

HMT shared LinkedIn post: "Everyone's debating whether PRDs should die. Wrong question."

**Core Argument:**
The spec didn't die. It **moved** in the workflow.

| Old Flow | New Flow |
|----------|----------|
| Idea → PRD → Design → Eng → QA → Ship | Idea → 5 prototypes → Evaluate → Kill 4 → Spec the survivor → Ship |
| 8-12 weeks | 1-2 weeks |
| PRD = permission document | PRD = decision record |
| Spec at step 2 | Spec at step 6 |

**Real-world examples:**
- **Anthropic (Boris Cherny):** No PRDs. Prototype in parallel, ship 20-30 PRs/day
- **OpenAI:** Still writes specs — 800M MAU need behavior contracts
- **Enterprises (5,000+ people):** Need docs for cross-timezone alignment

**The insight:**
> "The universal shift is that the spec comes AFTER you've touched working software."

**What the spec becomes:**
- Prototype shows **what**
- Spec explains **why**, **how you'll measure**, and **when you'll pull the plug**
- "Those three things separate a PM from a vibe coder"

**The punchline:**
> "The PMs prototyping first are shipping 5x more validated features. The PMs writing specs first are producing better documents about worse ideas."

### Connection to Autoresearch

This reinforces the morning synthesis:

| Before | After |
|--------|-------|
| Human designs experiment | Human defines success metric |
| Human runs test | Agent runs 100 tests |
| Human analyzes results | Agent keeps/discards automatically |
| Human iterates | Human curates and steers |

**The founder's job shifts from EXECUTOR to EVALUATOR:**
1. Define the scoring function (what does "good" look like?)
2. Create conditions for rapid prototyping
3. Curate winners and explain *why* they won
4. Kill faster than you ship
