Executive Brief — March 2026

What Is Autoresearch?

Autoresearch is a workflow pattern for autonomous experimentation. An AI agent reads a goal, proposes changes, runs experiments, keeps what works, discards what doesn't — and repeats overnight without human involvement.

Andrej Karpathy (former Tesla AI lead, OpenAI co-founder) open-sourced the pattern in March 2026. His repo hit 47,000 GitHub stars in one week.

The Paradigm Shift

Old model: Write PRD → Build → Test → Iterate (30 experiments/year)
New model: Define metric → Agent experiments overnight → Curate winners (36,500/year)

The spec didn't die — it moved. Instead of writing detailed PRDs upfront, PMs now define what "good" looks like (the evaluation function), and agents explore the solution space.

The new constraint isn't "can we build this?" — it's "can we measure success?"

Why This Matters for STAGE

STAGE already has the ingredients:

  • Clear metrics — TCR, LTV, chat depth, watch time
  • High volume — Millions of users, thousands of content pieces
  • Experiment culture — Full Funnel Sprint is already prototype-first

What's missing is the autonomous loop: automated eval → agent iteration → overnight experiments → morning curation.

The Opportunity

Character boards alone: ~150 experiments/night at ~$80/night. Within 4 weeks, the system generates a research journal of encoded knowledge:

"Bhojpuri audiences respond 22% better to characters that initiate storytelling vs. wait for questions."

No competitor can buy this. It comes from running the loop on your users, with your content, at your scale.

Risks & Guardrails

Primary risk: Optimizing the wrong metric (Goodhart's Law). The agent will find ways to game any proxy metric.

Mitigation: Seven guardrails built into the architecture:

  1. Composite metrics with quality gates
  2. Bounded mutations (prevent wild swings)
  3. Cross-character diversity constraints
  4. Proxy-true correlation monitoring (pause if chat depth stops predicting retention)
  5. Constitutional checks (LLM screens for manipulation)
  6. Red-team adversarial testing
  7. Human-in-the-loop checkpoints

The One-Line Summary

Autoresearch compresses the cost of iteration to near-zero. The new constraint is: can you write an evaluator that captures what actually matters?

Recommended Next Steps
